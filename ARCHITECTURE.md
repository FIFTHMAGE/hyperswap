# HyperSwap Architecture

## Structure

- `/src/core` - Core configuration and contexts
- `/src/infrastructure` - External services (API, storage, cache)
- `/src/shared` - Reusable components, hooks, utilities
- `/src/features` - Feature modules (swap, liquidity, portfolio, wallet)

## Principles

- Feature-based organization
- Colocation of related code
- Clear separation of concerns
